LuminaOS & Luce

Luce — Italian for "light" — is the design language of LuminaOS, a precision agriculture control platform for vertical farms and greenhouses.

The name carries a deliberate irony: a dark UI named after light. In LuminaOS, light is information — every colored pixel is a signal emerging from the void. In a greenhouse, light is literally the primary control variable: PPFD levels, photoperiod schedules, spectral composition. The name reflects both the design philosophy (dark canvas where data shines) and the domain (light-driven agriculture).

Mission & Audience

LuminaOS serves growers, agronomists, and facility operators who need instrument-grade precision in their controls. The UI must communicate complex environmental data — temperature, humidity, VPD, CO2, light spectra, pH, EC — clearly and instantly, like reading a cockpit instrument panel.

GROWERS Daily operators monitoring crops and adjusting environmental setpoints
AGRONOMISTS Technical specialists designing growth programs and analyzing yield data
OPERATORS Facility managers overseeing multiple chambers with fleet-level dashboards

Design Priorities

  • Glanceability — critical data visible in under 2 seconds
  • Precision — monospace values, units always shown, no rounding ambiguity
  • Status clarity — green/orange/red system immediately communicates health
  • Low eye strain — dark theme for control room and greenhouse monitoring

Design Inspiration

The three-materials metaphor (Void / Glass / Aluminum) draws from automotive instrument design, particularly the Ferrari Luce EV interior by Jony Ive / LoveFrom. This is inspiration, not imitation — LuminaOS adapts these principles for precision agriculture, not luxury cars.

●
Void = Darkness
Instrument panel black. Data emerges from absence of light.
◊
Glass = Depth
Translucent layers create spatial hierarchy without opacity.
◉
Aluminum = Touch
Machined metal feel for interactive controls. Detent precision.

Translation to Digital

  • Spring easing = physical detent feel of a machined dial
  • Gold accent = precision engineering indicator (active state, not decoration)
  • Void background = instrument panel darkness for maximum data contrast
  • Backdrop blur = glass layers suggesting physical depth between surfaces

Two Themes

Luce ships with two themes. Same components, same tokens, different personality.

Dark (Luce)
The default. Void/Glass/Aluminum materials, gold accent, glass blur panels. Designed for monitoring environments — control rooms and greenhouses — where dark UI reduces eye strain and makes data stand out.
Light
Clean white/teal aesthetic for general-purpose contexts — office use, documentation, reports. Solid surfaces (no glass blur), teal primary accent instead of gold. Same component structure, inverted token values.

Implementation

Light theme is activated via [data-theme="light"] on the <html> element. The same CSS custom property names are reused, so all component CSS works unchanged. See tokens.css for the full token mapping.

When to Use Each

  • Dark: Real-time monitoring, greenhouse control rooms, night shifts, immersive dashboards
  • Light: Office environments, printed reports, documentation, daytime casual use
  • Let the user choose — never force a theme based on ambient light or time of day

Voice & Tone

Technical, precise, confident, understated. LuminaOS speaks like a well-calibrated instrument — it reports facts, not feelings.

Status Messages
pH 6.2 — Target: 6.0
Terse. Value + context. No "Your pH is currently..."
Error Messages
Sensor offline. Check connection.
Direct. Problem + action. No "Oops!" or "Something went wrong."
AI Recommendations
VPD trending high. Consider reducing 1–2°C.
Measured. Observation + suggestion. Never imperative ("You must...").
Empty States
No devices found. Add a device to get started.
Neutral. State + next step. No "It's lonely in here!" or emoji.

Writing Rules

  • Always use units — 24.2°C, not 24.2. 420 µmol, not 420.
  • Monospace for data — all values in JetBrains Mono
  • Uppercase labels — section headers, form labels, column headers
  • No emoji — ever. Not in status, not in errors, not in success messages.
  • No casual language — no "Great!", "Awesome!", "Oops!", "Hang tight!"
  • Abbreviations OK — Temp, RH, PPFD, VPD, EC. Users know them.

Size Minimums

  • Icon: 32px minimum (app icon at smallest launcher size)
  • Wordmark: 120px minimum width (legibility threshold)
  • Clear space: 1x the height of the logo on all sides

Theme Context

  • Dark theme: white wordmark, clear icon
  • Light theme: black wordmark, solid icon
  • Never place a white logo on a light background or vice versa

Do's and Don'ts

Use correct logo for theme
White wordmark on dark, black wordmark on light. Match the icon variant (clear/solid) to the background.
Mismatched logo/background
Never place the white logo on white or light gray. Never place the black logo on dark backgrounds. Always maintain high contrast.
Technical voice
"Sensor offline. Check connection." Direct, factual, actionable. Let the data speak.
Casual voice
"Oops! Looks like your sensor took a nap. Give it a nudge?" No whimsy, no anthropomorphization, no emoji.
Gold/teal = active state
Primary accent indicates the active, selected, or primary action element. It carries meaning: "this is where your attention should be."
Gold/teal as decoration
Never use the primary accent for borders, backgrounds, or visual flair. If everything is gold, nothing is.
Respect theme boundaries
Use the three-materials metaphor (Void/Glass/Aluminum) in dark mode. Use solid surfaces in light mode. Each theme has its own visual language.
Mix theme elements
Don't use glass blur in light mode or solid white cards in dark mode. Each theme is internally consistent — mixing creates visual confusion.
